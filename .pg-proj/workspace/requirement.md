# 简化安装脚本使用方式（Phase 1 输出）

## 目标

将安装入口简化为“网好时一条命令可执行”，同时保留仓库内可审计脚本方式。

## 用户确认输入（采访结论）

1. 最简入口：
   - 首次支持 `curl ... | sh`
   - 后续支持更便捷方式（通过包装参数）
2. 默认行为：
   - 先快速检测摘要，再自动继续安装
3. 交互策略：
   - 全默认无交互
   - 仅 Windows WSL 推荐保留确认
4. 交付形式：
   - 提供短命令包装（如 `--fast`）
   - 提供可部署的脚本入口模板

## 成功标准

- 用户可通过一条命令执行安装（fast 模式）
- fast 模式默认自动推进，无需反复输入
- README 提供最短可复制命令
- 提供后续部署时可直接复用的入口模板
