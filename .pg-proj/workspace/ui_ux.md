# CLI 交互设计（全量检测 + Windows WSL2 推荐）

## 1. 主流程

1. 输出欢迎信息与模式（strict mode）
2. 环境识别（OS/架构/权限）
3. Windows 分支：
   - 显示“推荐 WSL2 安装”提示
   - 询问：`是否现在安装 WSL2? [Y/n]`
   - 选择 Y：执行 `wsl --install -d Ubuntu`，提示重启后结束
   - 选择 n：继续原生 Windows 路径
4. 全量检测阶段（跨平台）
5. 自动修复阶段
6. 官方安装器执行
7. 最终验证与摘要

## 2. 交互文案关键点

- 所有失败场景必须输出：
  - 错误码
  - 原因简述
  - 可复制修复命令
- Windows WSL 推荐必须明确“推荐但可跳过”
- 检测结果按表格/分组输出（PASS/WARN/FAIL）

## 3. 状态

- PASS：满足要求
- WARN：非阻断，建议修复
- FAIL：阻断（严格模式下立即中断）
